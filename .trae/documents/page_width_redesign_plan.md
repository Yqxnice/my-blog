# 页面主体宽度重新设计方案

## 项目概述
本项目旨在重新设计博客网站的所有页面主体宽度，包括导航栏、内容区域和底部，以提供更一致和优化的用户体验。

## 当前问题分析
1. 导航栏使用了 `max-w-[70%]` 的固定百分比宽度，在大屏幕上可能显得过窄
2. 页面内容区域使用 `container mx-auto`，但不同页面可能有不一致的边距设置
3. 底部区域宽度为 `max-w-4xl`，与其他部分可能不够协调
4. 缺乏统一的宽度规范，导致各页面宽度不一致

## 设计目标
1. 建立统一的响应式宽度规范
2. 提升跨设备的一致性体验
3. 优化主要内容区域的可读性
4. 保持现代化的设计感

## 具体实施方案

### 1. 全局宽度规范定义
定义以下断点处的最大宽度：
- xs (320px): 100% 宽度，左右16px边距
- sm (640px): 100% 宽度，左右24px边距
- md (768px): 100% 宽度，最大750px，左右居中
- lg (1024px): 100% 宽度，最大1000px，左右居中
- xl (1280px): 100% 宽度，最大1200px，左右居中
- 2xl (1536px): 100% 宽度，最大1400px，左右居中

### 2. 导航栏宽度调整
**当前**: `max-w-[70%]`
**目标**: 
- Mobile: 100% 宽度，左右16px边距
- Tablet: 100% 宽度，最大750px，左右居中
- Desktop: 100% 宽度，最大1000px，左右居中
- Large screen: 100% 宽度，最大1200px，左右居中

### 3. 内容区域宽度调整
**当前**: `container mx-auto px-4 py-12`
**目标**: 
- Mobile: 100% 宽度，左右16px边距
- Tablet: 100% 宽度，最大750px，左右居中
- Desktop: 100% 宽度，最大1000px，左右居中
- Large screen: 100% 宽度，最大1200px，左右居中

### 4. 底部宽度调整
**当前**: `max-w-4xl mx-auto px-4 sm:px-6 lg:px-8`
**目标**:
- Mobile: 100% 宽度，左右16px边距
- Tablet: 100% 宽度，最大750px，左右居中
- Desktop: 100% 宽度，最大1000px，左右居中
- Large screen: 100% 宽度，最大1200px，左右居中

## 实施步骤

### 步骤1: 更新Tailwind配置
- 添加自定义容器宽度类

### 步骤2: 修改导航栏组件
- 更新 `components/blog/Navbar.tsx` 中的宽度设置

### 步骤3: 修改布局组件
- 更新 `app/(blogs)/layout.tsx` 中的内容区域

### 步骤4: 修改各页面组件
- 更新 `app/(blogs)/page.tsx`
- 更新 `app/(blogs)/blogs/page.tsx`
- 更新 `app/(blogs)/blogs/[id]/page.tsx`

### 步骤5: 修改底部组件
- 更新 `components/blog/Footer.tsx` 中的宽度设置

### 步骤6: 测试和验证
- 在各种设备和屏幕尺寸上测试
- 确保所有页面宽度一致
- 验证可读性和可用性

## 预期效果
1. 统一的视觉体验：所有页面宽度保持一致
2. 优化的阅读体验：主要内容区域宽度适中
3. 响应式兼容：在各种设备上都有良好表现
4. 现代化设计：符合当前网页设计趋势
