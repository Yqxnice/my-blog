## 问题描述

在使用 Next.js 16 的 `useRouter` 进行路由跳转时，发现使用 `router.push()` 跳转到动态路由页面时，页面没有正确渲染。

## 复现步骤

1. 在组件中使用 `useRouter()` 获取路由实例
2. 调用 `router.push('/blogs/[id]', `/blogs/${blogId}`)` 进行跳转
3. 页面跳转到正确的 URL，但内容没有更新

## 解决方案

经过排查，发现问题出在路由缓存机制上。需要在 `router.push()` 中添加 `{ scroll: false, shallow: false }` 选项：

```typescript
router.push('/blogs/[id]', `/blogs/${blogId}`, { scroll: false, shallow: false });
```

这样可以强制刷新页面内容，确保动态路由参数正确传递。

## 原因分析

Next.js 16 为了优化性能，对路由跳转进行了缓存处理。当跳转到相同路由结构的不同参数页面时，默认会使用缓存的组件实例，导致内容不更新。

通过设置 `shallow: false`，可以强制重新获取数据，确保页面内容与 URL 参数保持一致。
